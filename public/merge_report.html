推荐合并策略
  • 1. 标准合并 (Standard/Recursive)

    默认策略。尝试自动合并，遇到冲突时保留双方修改标记 (<<<<<<<)，需人工介入解决。

  • 2. 优先使用传入更改 (Theirs)

    对于冲突部分，直接使用 的代码，丢弃当前分支的修改。

    git merge -X theirs
  • 3. 优先保留当前更改 (Ours)

    对于冲突部分，强制保留 的代码，忽略传入分支的修改。

    git merge -X ours
手动解决指引
  1. 打开终端，进入仓库目录：
    cd /path/to/repo
  2. 检出目标分支：
    git checkout
  3. 拉取最新代码（防止落后）：
    git pull origin
  4. 尝试合并源分支：
    git merge
  5. 此时 Git 会提示冲突。使用编辑器打开冲突文件，搜索 <<<<<<<，修改并保存。
  6. 提交更改：
    git add . && git commit -m "Merge branch ''"
