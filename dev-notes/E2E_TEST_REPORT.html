# 🔍 发现的问题

# 1. SSH 密钥权限问题

问题: 使用数据库 SSH 密钥克隆时报错

git@github.com: Permission denied (publickey).
1

原因:

  • 数据库中存储的密钥可能不完整
  • 或者密钥格式有问题

解决方案:

  1. 使用系统默认密钥（不选择凭证）
  2. 使用 HTTPS + Token
  3. 重新添加正确的 SSH 密钥到数据库

# 2. API 路由命名

观察:

  • 注册使用 /repo/create (单数)
  • 列表使用 /repo/list (单数)

建议: 保持一致性，目前是正确的
