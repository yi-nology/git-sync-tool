## Project Overview

`git-manage-service` is a lightweight multi-repository Git management and automated sync system. It exposes:
- An HTTP API (CloudWeGo Hertz, default port 8080)
- An RPC service (CloudWeGo Kitex, default port 8888)
- An MCP TCP server (port 9000) for AI tool integration
- A Wails desktop app (embeds the frontend)

## Build & Development Commands

```bash
# Full build (frontend + backend)
make build-full

# Backend only
make build

# Run all tests
make test

# Run a single test (example)
go test -v ./biz/service/git/... -run TestFunctionName

# Format code
make fmt

# Lint (requires golangci-lint)
make lint

# Build frontend and sync to pkg/embed for binary embedding
make build-frontend-integrate

# Multi-platform release builds
make build-all

# Desktop app build
make desktop
```

**Important**: `CGO_ENABLED=1` is required (SQLite dependency). Ensure a C compiler is available.

## Architecture

### Request Flow
```
Client Ôćĺ Router Ôćĺ Handler Ôćĺ Service Ôćĺ DAO Ôćĺ Database
```

### Key Layers

| Layer | Location | Responsibility |
|-------|----------|----------------|
| Router | `biz/router/` | Route registration and middleware wiring |
| Handler | `biz/handler/{module}/` | HTTP param binding, validation, response wrapping |
| Service | `biz/service/` | Business logic and orchestration |
| DAO | `biz/dal/db/` | GORM-based CRUD operations |
| Model (DB) | `biz/dal/model/` | GORM struct definitions |
| Model (Proto) | `biz/model/` | Protobuf-generated request/response types ÔÇö **do not edit manually** |

### Entry Points

- `cmd/server/main.go` ÔÇö Web service (supports `--mode=all|http|rpc`)
- `main.go` ÔÇö Wails desktop app entry point

### MCP Service

`biz/mcp/` implements an MCP TCP server with 13+ tools (git_clone, git_push, git_log, sync tasks, notifications, etc.). Used for external AI tool integrations.

## Code Generation

API types in `biz/model/` and route stubs in `biz/router/*/*.go` are generated from Protobuf IDL files (`idl/`). **Never manually edit files marked with:**
```go
// Code generated by hz. DO NOT EDIT.
```

To regenerate after IDL changes:
```bash
hz update -idl idl/biz/*.proto
```

Safe to edit: `biz/handler/*/`, `biz/service/`, `biz/dal/`, `biz/router/register.go`.

## Configuration

Main config: `conf/config.yaml`
- Database: SQLite by default (`data/git_sync.db`), also supports MySQL/PostgreSQL
- HTTP port default: 12345 (overridable)
- RPC port default: 8888

## Code Style (from AGENT.md)

- **Imports**: Three-group ordering ÔÇö stdlib, third-party, internal
- **Struct tags**: `gorm` Ôćĺ `json` Ôćĺ `form` Ôćĺ `binding` order
- **Error wrapping**: Use `fmt.Errorf("op: %w", err)` to preserve stack context
- **Context keys**: Define typed `contextKey` constants (never use raw strings)
- **Comments**: Function-level in English; inline business logic comments may be in Chinese
- **Proto naming**: Messages use PascalCase; fields use snake_case; request/response pattern: `{Action}{Entity}Request` / `{Entity}Response`

## Frontend

Located in `frontend/` (Vue 3 + TypeScript + Element Plus). Built artifacts are embedded into the binary via `pkg/embed/`. Always run `make build-frontend-integrate` before building the backend if frontend changes were made.
