# AGENTS.md

Quick-reference for agents working in this repo. See `CLAUDE.md` for the full project overview and `AGENT.md` for detailed coding standards.

## Build Commands

```bash
make build-full              # Frontend + backend (recommended for full builds)
make build                   # Backend only
make build-frontend-integrate # Build frontend, copy to public/ and pkg/embed/
make test                    # Run all tests
go test -v ./biz/service/git/... -run TestFunctionName  # Single test
make fmt                     # Format Go code
make lint                    # Lint (requires golangci-lint)
```

**CRITICAL**: `CGO_ENABLED=1` is required — the project uses SQLite via a C driver. A C compiler must be available. The Makefile exports this automatically; if running `go build` directly, set it yourself.

**Build order for frontend changes**: `make build-frontend-integrate` → `make build`. The frontend build must complete before the backend can embed it.

## Two Entry Points

- `cmd/server/main.go` — Web/RPC server (build tag `!desktop`). Accepts `--mode=all|http|rpc`.
- `main.go` — Wails desktop app (build tag `desktop`). Embeds `frontend/dist` via `//go:embed`.

They share the same `biz/` backend but are compiled separately due to build tags.

## Architecture

```
Request: Client → Router → Handler → Service → DAO → Database (GORM/SQLite)
```

| Layer | Path | Notes |
|---|---|---|
| Router | `biz/router/{module}/` | Generated by Hz — do NOT edit except `register.go` |
| Handler | `biz/handler/{module}/` | HTTP binding + validation — safe to edit |
| Service | `biz/service/{module}/` | Business logic — safe to edit |
| DAO | `biz/dal/db/` | GORM CRUD — safe to edit |
| DB Model | `biz/dal/model/` | GORM structs — safe to edit |
| Proto Model | `biz/model/` | Generated from `idl/` — do NOT edit |
| MCP Server | `biz/mcp/` | TCP server on port 9000, 13+ tools for AI integration |

## Code Generation (Hz)

API types and route stubs are generated from Protobuf IDL files in `idl/biz/*.proto`.

```bash
hz update -idl idl/biz/*.proto      # Regenerate all
hz update -idl idl/biz/repo.proto    # Single module
```

**Never edit** files with `// Code generated by hz. DO NOT EDIT.` — this includes `biz/model/**/*.go` and `biz/router/*/*.go` (except `register.go`).

## Style Conventions

- **Imports**: Three groups — stdlib / third-party / internal
- **Struct tags**: `gorm` → `json` → `form` → `binding` order
- **Error wrapping**: `fmt.Errorf("op: %w", err)` to preserve stack
- **Context keys**: Typed `contextKey` constants, never raw strings
- **Comments**: Function-level in English; inline business logic may be in Chinese
- **Proto naming**: Messages PascalCase; fields snake_case; pattern: `{Action}{Entity}Request` / `{Entity}Response`
- **No comments** in code unless explicitly requested

## Configuration

`conf/config.yaml` — SQLite default (`data/git_sync.db`), HTTP port 12345, RPC port 8888.

## Frontend

`frontend/` — Vue 3 + TypeScript + Element Plus + Vite. Built artifacts go to `pkg/embed/` for binary embedding via `pkg/embed/`. Always run `make build-frontend-integrate` before `make build` if frontend changed.

## CI / Release

- **Server release**: Push `v*` tag → `.github/workflows/release.yml` builds 6 platform binaries. Note: CI uses `CGO_ENABLED=0` for the server release, so the SQLite driver must support that (or use the build script).
- **Desktop release**: Push `v*-desktop` tag → `.github/workflows/desktop.yml` builds macOS/Windows/Linux via Wails.

## Module Map

`biz/handler/`, `biz/router/`, and `biz/service/` are organized by domain: `repo`, `branch`, `commit`, `file`, `tag`, `stash`, `patch`, `sync`, `notification`, `credential`, `sshkey`, `audit`, `stats`, `spec`, `system`, `version`, `webhook`, `submodule`.

Each domain typically has: a `.proto` in `idl/biz/`, a handler in `biz/handler/{domain}/`, a service in `biz/service/{domain}/`, and a DAO in `biz/dal/db/`.

## Gotchas

- The `.gitignore` now excludes `pkg/embed/`, `public/`, `build/`, `data/git_sync.db`, and `docs-site/node_modules/`. Do not re-add these.
- `docs-site/` is a separate docs website with its own `node_modules` — not the main frontend.
- `biz/kitex_gen/` and `biz/kitex_info.yaml` are Kitex RPC generated code.
- Database migrations live in `biz/dal/db/migration.go`.
