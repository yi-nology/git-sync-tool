# Git Manage Service - 产品文档

## 1. 产品简介

**Git Manage Service** 是一款专为研发工程师和 DevOps 设计的轻量级 Git 同步管理系统。它解决了多仓库、多分支之间代码同步繁琐、缺乏记录、难以自动化的痛点。通过统一的 Web 界面，用户可以轻松配置同步规则，实现代码在不同远程仓库之间的自动化流转。

## 2. 核心功能

### 2.1 仓库管理

![仓库详情](./images/repo-detail.png)

- **本地仓库注册**：支持注册服务器上的本地 Git 仓库路径。
- **仓库克隆**：支持从远程仓库克隆到本地。
- **自动检测**：注册时自动校验路径是否为有效的 Git 仓库。
- **远程仓库管理**：支持添加、编辑、删除远程仓库配置。

### 2.2 多仓同步管理

![同步任务](./images/sync-tasks.png)

- **灵活规则**：支持定义 `源仓库/Remote/分支` 到 `目标仓库/Remote/分支` 的同步流向。
- **定时同步**：内置 Cron 调度器，支持 Cron 表达式（如 `0 2 * * *`）实现自动化周期同步。
- **手动触发**：支持一键立即执行多仓同步。
- **高级选项**：支持配置 `git push` 参数（如 `--force`, `--no-verify`）。
- **任务编辑**：支持随时调整现有任务的配置信息。
- **全分支同步**：支持同步仓库的所有分支。

## 3. 技术架构

### 3.1 后端
- **语言**: Go 1.25.0
- **Web 框架**: CloudWeGo Hertz
- **RPC 框架**: CloudWeGo Kitex
- **数据库**: SQLite, MySQL, PostgreSQL

- **对象存储**: MinIO (可选)

- **路由**: Vue Router
- **HTTP 客户端**: Axios
- **图表**: ECharts
- **代码编辑器**: Monaco Editor
- **差异对比**: diff2html
- **构建工具**: Vite
- **语言**: TypeScript

## 4. 应用场景

### 4.1 代码备份
- **场景**：定期将代码从主仓库同步到备份仓库，确保代码安全。
- **配置**：设置定时同步任务，每天或每周执行一次。
- **通知**：配置邮件通知，当同步失败时及时收到提醒。

### 4.2 CI/CD 集成
- **场景**：当 CI/CD 流程完成后，自动将代码同步到生产环境仓库。
- **配置**：使用 Webhook 触发同步，确保代码在构建完成后及时同步。
- **通知**：配置企业微信通知，当同步完成时通知相关人员。
