# Git 同步功能重新设计方案

## 需求分析

### 同步场景
1. **全仓库同步** - 一键同步所有已配置的同步规则
2. **跨仓库同步** - 仓库 A 的某个分支同步到仓库 B
3. **本地 ↔ 远端** - 本地推送或远端拉取
4. **特殊 Git 命令** - `--force`, `--tags`, `--prune`, `--no-verify` 等

### 同步方向
| 模式 | 描述 |
|------|------|
| Push | 本地 → 远端 |
| Pull | 远端 → 本地 |
| Mirror | 双向镜像同步 |
| Cross-Repo | 跨仓库同步 |

## 技术实现

### 后端扩展

```go
// SyncMode 同步模式
type SyncMode string

const (
    SyncModePush      SyncMode = "push"       // 本地 → 远端
    SyncModePull      SyncMode = "pull"       // 远端 → 本地
    SyncModeMirror    SyncMode = "mirror"     // 双向镜像
    SyncModeCrossRepo SyncMode = "cross-repo" // 跨仓库
    SyncModeAllBranch SyncMode = "all-branch" // 全分支
)

// GitOptions Git 选项
type GitOptions struct {
    Tags     bool `json:"tags"`     // --tags
    Force    bool `json:"force"`    // --force
    Prune    bool `json:"prune"`    // --prune
    NoVerify bool `json:"no_verify"` // --no-verify
    DryRun   bool `json:"dry_run"`   // --dry-run
}
```

### API 扩展

```
POST /api/v1/sync/preview    # 预览同步变更
POST /api/v1/sync/batch      # 批量执行同步
GET  /api/v1/repos           # 获取仓库列表 (用于跨仓库选择)
```

## 风险提示

- `--force` 强制推送可能覆盖远端提交
- `--prune` 可能删除本地分支引用
- 跨仓库同步需要确保目标仓库权限
