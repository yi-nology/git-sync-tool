# 使用指南

## 1. 系统概览

Git Manage Service 是一个轻量级的多仓库、多分支自动化同步管理系统，提供了友好的 Web 界面，支持定时任务、Webhook 触发、多渠道消息通知以及详细的同步日志记录。

### 主要功能

- **多仓库管理** - 轻松注册和管理本地 Git 仓库
- **灵活同步规则** - 支持任意 Remote 和分支之间的同步
- **自动化执行** - 内置 Cron 调度器，支持定时同步
- **Webhook 集成** - 支持通过外部系统触发同步
- **安全可靠** - 冲突检测、Fast-Forward 检查及 Force Push 保护
- **多渠道通知** - 支持钉钉、企业微信、飞书、蓝信、邮件、自定义 Webhook
- **消息模板** - 支持 Go 模板语法，支持渠道级和事件级两层模板配置
- **SSH 密钥管理** - 支持将 SSH 密钥存储在数据库中，统一管理仓库认证
- **代码质量分析** - 集成 lint 规则，提供代码质量检查
- **可视化界面** - 提供直观的 Web UI，查看历史、日志及管理任务

## 2. 快速开始

### 2.1 启动服务

```bash
# 仅启动 HTTP 服务
./git-manage-service --mode=http

# 仅启动 RPC 服务
./git-manage-service --mode=rpc

# 同时启动 HTTP 和 RPC 服务（默认）
./git-manage-service --mode=all
```

### 2.2 访问界面

浏览器打开: [http://localhost:38080](http://localhost:38080)

## 3. 功能操作指南

### 3.1 仓库管理

#### 注册仓库
1. 点击导航栏的 **“仓库管理”**。
2. 点击 **“注册仓库”** 按钮。
3. 输入仓库 **名称**（便于识别）和 **本地绝对路径**（服务器上存在的 Git 仓库路径）。
4. 点击保存。

#### 克隆仓库
1. 点击导航栏的 **“仓库管理”**。
2. 点击 **“克隆仓库”** 按钮。
3. 输入 **仓库 URL**、**本地路径** 和 **名称**。
4. 选择 **认证方式**（SSH 密钥或用户名密码）。
5. 点击保存。

### 3.2 同步任务管理

#### 创建同步任务
1. 点击导航栏的 **“同步任务”**。
2. 点击 **“新建任务”**。
3. 填写配置：
    - **源仓库**：选择已注册的仓库。
    - **源 Remote/分支**：如 `origin` / `main`。
    - **目标 Remote/分支**：如 `backup` / `main`。
    - **Push 选项**（可选）：如需强制覆盖，可填 `--force`。
    - **Cron 表达式**（可选）：如 `*/10 * * * *` 表示每 10 分钟同步一次。留空则仅支持手动触发。
    - **启用**：勾选后 Cron 任务即刻生效。
4. 点击保存。

#### 执行同步任务
- **手动运行**：在任务列表中点击 **“运行”** 按钮，系统将在后台异步执行同步。
- **编辑任务**：点击 **“编辑”** 按钮可修改任务配置。
- **删除任务**：点击 **“删除”** 按钮可删除任务。

### 3.3 分支管理

1. 点击导航栏的 **“分支管理”**。
2. 选择一个仓库查看其分支列表。
3. 支持的操作：
    - **切换分支**：点击分支名称旁的 **“切换”** 按钮。
    - **推送分支**：点击分支名称旁的 **“推送”** 按钮。
    - **拉取分支**：点击分支名称旁的 **“拉取”** 按钮。
    - **创建分支**：点击 **“创建分支”** 按钮。
    - **删除分支**：点击分支名称旁的 **“删除”** 按钮。

### 3.4 版本历史管理

1. 点击导航栏的 **“版本历史”**。
2. 选择一个仓库查看其标签列表。
3. 支持的操作：
    - **创建标签**：点击 **“创建标签”** 按钮。
    - **推送标签**：点击标签名称旁的 **“推送”** 按钮。
    - **删除标签**：点击标签名称旁的 **“删除”** 按钮。

### 3.5 文件浏览

1. 点击导航栏的 **“文件浏览”**。
2. 选择一个仓库和分支查看其文件结构。
3. 点击文件名称查看文件内容。
4. 点击 **“历史”** 按钮查看文件的修改历史。

### 3.6 审计日志

1. 点击导航栏的 **“审计日志”**。
2. 查看所有操作日志，支持按类型、对象、时间筛选。
3. 点击 **“查看详情”** 按钮查看日志详情。

### 3.7 系统设置

#### SSH 密钥管理
1. 点击导航栏的 **“系统设置”**。
2. 点击 **“SSH 密钥”** 标签。
3. 点击 **“添加密钥”** 按钮添加新的 SSH 密钥。
4. 填写 **名称** 和 **私钥内容**。
5. 点击保存。

#### 通知渠道管理
1. 点击导航栏的 **“系统设置”**。
2. 点击 **“通知渠道”** 标签。
3. 点击 **“添加渠道”** 按钮添加新的通知渠道。
4. 选择 **渠道类型**（钉钉、企业微信、飞书、蓝信、邮件、自定义 Webhook）。
5. 填写渠道配置信息。
6. 点击保存。

#### 全局 Git 配置
1. 点击导航栏的 **“系统设置”**。
2. 点击 **“Git 配置”** 标签。
3. 填写 **用户名** 和 **邮箱**。
4. 点击保存。

## 4. 高级功能

### 4.1 代码质量分析

1. 点击导航栏的 **“代码质量”**。
2. 选择一个仓库查看其代码质量分析结果。
3. 查看代码质量报告，包括代码行数、注释行数、空白行数等。

2. 选择一个仓库查看其 Git 有效提交度量结果。
3. 查看贡献者排行、提交趋势、文件类型分布等分析。

### 4.3 真实工程代码度量

1. 点击导航栏的 **“工程度量”**。
2. 选择一个仓库查看其真实工程代码度量结果。
3. 基于 git blame 分析代码归属，统计代码行数、注释行数、空白行数等。

## 5. Webhook 集成

外部系统可通过 HTTP POST 请求触发多仓同步。

- **URL**: `http://<server>:38080/api/webhooks/task-sync`
- **Method**: `POST`
- **Headers**:
    - `Content-Type`: `application/json`
    - `X-Hub-Signature-256`: `sha256=<HMAC-SHA256 Signature>`
- **Body**: `{"task_id": 1}`

*注：详细 Webhook 开发文档请参考 `docs/webhook.md`。*

## 6. 最佳实践

### 6.1 同步策略

- **定时同步**：对于重要的代码库，建议设置定时同步任务，确保代码及时备份。
- **Webhook 触发**：对于 CI/CD 流程，建议使用 Webhook 触发同步，确保代码在构建完成后及时同步。
- **多渠道通知**：建议配置多个通知渠道，确保同步失败时能够及时收到通知。

### 6.2 安全建议

- **SSH 密钥管理**：使用系统提供的 SSH 密钥管理功能，避免在配置文件中明文存储密钥。
- **Webhook 密钥**：设置强密码作为 Webhook 密钥，避免未授权访问。
- **IP 白名单**：如需限制 Webhook 访问来源，可配置 IP 白名单。

### 6.3 性能优化

- **仓库数量**：建议不要注册过多仓库，以免影响系统性能。
- **同步频率**：根据代码变更频率设置合理的同步频率，避免过于频繁的同步。
- **数据库选择**：对于大型部署，建议使用 MySQL 或 PostgreSQL 数据库，而非 SQLite。

## 7. 故障排查

### 7.1 同步失败

- **检查网络连接**：确保服务器能够访问远程仓库。
- **检查权限**：确保服务器有足够的权限访问和修改仓库。
- **检查 SSH 密钥**：确保 SSH 密钥配置正确，且有权限访问远程仓库。
- **查看日志**：查看同步任务的详细日志，了解失败原因。

### 7.2 系统启动失败

- **检查配置文件**：确保配置文件格式正确，且所有必要的配置项都已设置。
- **检查端口占用**：确保系统使用的端口（默认 38080）未被其他进程占用。
- **检查数据库**：确保数据库连接正常，且有足够的权限。

### 7.3 通知发送失败

- **检查渠道配置**：确保通知渠道的配置信息正确。
- **检查网络连接**：确保服务器能够访问通知服务的 API。
- **查看日志**：查看通知发送的详细日志，了解失败原因。

## 8. 常见问题

### 8.1 如何添加新的远程仓库？

1. 进入仓库详情页面。
2. 点击 **“远程仓库”** 标签。
3. 点击 **“添加远程”** 按钮。
4. 填写 **名称** 和 **URL**。
5. 点击保存。

### 8.2 如何修改同步任务的 Cron 表达式？

1. 进入同步任务列表页面。
2. 找到要修改的任务，点击 **“编辑”** 按钮。
3. 修改 **Cron 表达式** 字段。
4. 点击保存。

### 8.3 如何查看同步任务的执行历史？

1. 进入同步任务列表页面。
2. 找到要查看的任务，点击 **“历史”** 按钮。
3. 查看该任务的执行历史记录。
4. 点击 **“查看日志”** 按钮查看详细的执行日志。

### 8.4 如何配置邮件通知？

1. 进入系统设置页面。
2. 点击 **“通知渠道”** 标签。
3. 点击 **“添加渠道”** 按钮。
4. 选择 **渠道类型** 为 **邮件**。
5. 填写 **SMTP 服务器**、**端口**、**用户名**、**密码**、**发件人** 等信息。
6. 点击保存。
